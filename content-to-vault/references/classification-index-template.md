# Classification Index 初始化模板

仅当 `{VAULT}/Library/_system/classification_index.md` 不存在、内容不足、明显过旧或无法支持当前分类判断时读取本文件。正常归档不要读取。

## 初始内容

```markdown
# 分类索引

## 分类原则

- 目录按未来检索意图划分，不按来源平台、文件格式或关键词堆叠划分。
- 优先复用已有目录。
- 如果没有合适目录，创建宽泛目录。
- 默认使用两级目录。
- 不主动创建三级目录。
- tags 保持 3-6 个、优先复用已有标签。细粒度概念放 topics。

## 已有目录

H3 为一级目录，H4 为二级目录。每条附一句描述说明该目录的内容范围与边界。

<!-- agent 逐步追加，格式：
### 一级分类名
用于该分类的内容范围与边界说明。

#### 二级分类名
该二级分类的具体内容范围。
-->

## 已有 Tags

按一级分类组织。agent 归档时：先生成 3-6 个候选 tag → 到对应分类下查找语义接近的已有 tag → 有则替换，无则追加到对应分类下。

<!-- agent 逐步追加，格式：
### 一级分类名
- tag1
- tag2
- ...
-->
```

## 新建目录规则

需要创建新目录时（即一级或二级目录都没有可复用的匹配项），按以下规则约束新建目录的宽度和命名。

### 不准用于主目录的内容

不要用以下内容创建主目录：

```text
来源平台
文件格式
单个产品
单个公司
单个地点
单个人名
短期热点
过细关键词
```

例外：如果某个名称已经存在于 `classification_index.md`，或明显是用户长期主题，可以复用。

## 新增目录代码模板

使用 `execute_code` 一次性完成读取→修改→写回。**禁止用 `patch`**（实测 old_string 匹配频繁失败，平均浪费 2-3 turns）。

```python
index_path = f"{vault}/Library/_system/classification_index.md"
with open(index_path) as f:
    content = f.read()

# 新增目录：在"## 已有目录"末尾追加 H3/H4 条目
# 新增 tags：在对应一级分类的 tags 列表末尾追加
# ...（按实际需要修改 content）

with open(index_path, 'w') as f:
    f.write(content)
```

新增目录格式（H3 和 H4 都必须附一句描述）：

```python
# 追加到 "## 已有目录" 段末尾
new_category_block = """
### 一级分类名
用于该分类的内容范围与边界说明。

#### 二级分类名
该二级分类的具体内容范围说明。
"""

# 追加 tags 到 "## 已有 Tags" 下对应一级分类的列表末尾
# 如果该一级分类不存在，新建 H3 段
new_tags_block = """
### 一级分类名
- tag1
- tag2
"""
```

规则：
- 如果创建了新的 L1/L2 目录组合，**H3 和 H4 都必须附一句描述**，说明该目录的内容范围与边界。不能只有 H4 有描述而 H3 没有（常见遗漏）。
- 将本次使用的 tags 对照「已有 Tags」：语义接近的替换为已有写法，没有的追加到对应一级分类下。
- tags 段必须用 H3 分类标题 + 列表格式，不能用纯 flat 列表。
- 不重写整个文件（只追加/修改必要行）。
